# Guide de Contribution 🎉

Merci de votre intérêt pour contribuer à ce projet ! 🙏

## Comment contribuer 🚀

1. **Fork** ce dépôt sur GitHub. 🍴

2. **Clone** votre fork localement :  

```
git clone https://github.com/votre-utilisateur/nom-du-projet.git
```

3. **Créez une branche** pour votre fonctionnalité ou correction de bug : 🌿

```
git checkout -b ma-nouvelle-fonctionnalite
```

4. **Faites vos modifications** en suivant les bonnes pratiques du projet. 🛠️

5. **Commitez vos changements** avec un message clair et précis : ✍️

```
git commit -m "Description claire de la modification"
```

6. **Poussez votre branche** sur votre fork : 🚀

```
git push origin ma-nouvelle-fonctionnalite
```

7. **Ouvrez une Pull Request (PR)** sur la branche principale du dépôt d'origine. 🔀✨

Règles de contribution
Respectez la convention de code du projet.

Documentez vos modifications si nécessaire.

Écrivez des tests pour toute nouvelle fonctionnalité ou correction.

Assurez-vous que votre code passe les tests existants.

Soyez poli et respectueux dans les échanges. 🤝

Types de contributions
Rapport de bugs : merci de fournir un maximum d’informations pour reproduire le problème.

Suggestions d’amélioration : n’hésitez pas à ouvrir une issue pour discuter.

Corrections et nouvelles fonctionnalités : via Pull Requests.

Communication
Pour toute question ou discussion, utilisez les issues GitHub du projet.

Merci encore pour votre contribution ! 🚀🙏
